# Feature Workflow (MCP-grounded)

Usage: `Apply workflow: .clinerules/workflows/feature.md`

- [ ] Use `sequential-thinking` to decompose the request into a step-by-step implementation plan.
- [ ] List acceptance criteria and get user confirmation.
- [ ] Use `research-expert` skill if implementation involves unfamiliar libraries or complex architecture.
- [ ] Use `code-navigator` skill to discover affected files and perform surgical implementation.
- [ ] Add or update tests for the new behavior.
- [ ] Run verification commands (`npm run lint`, focused tests).
- [ ] Use `git` MCP to review changes against the plan.
- [ ] Summarize changes, verification results, and residual risks.
